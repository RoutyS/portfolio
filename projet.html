Projet de deuxieme année

Context

Après deux années de réorganisations internes, tant au niveau du personnel que du fonctionnement administratif, l'entreprise GSB souhaite moderniser l'activité de visite médicale.

Expression de besoin

Fonctionnalité: Gestion des medicaments

le client doit pouvoir:
• Insérer, modifier, lister des types d’individus
• Insérer, modifier, lister des médicaments
• Insérer une prescription (pour un médicament, un type d’individu, un dosage)
• Créer pour un médicament, la liste des médicaments non conseillés.
• Avoir des statistiques sous forme d’indicateurs (nombres, pourcentages, …).

ressources utilisées

Pour la réalisation des maquettes c’est fait sur Figma.
PhpMyAdmin pour gérer la base de données.
Pour coder et construire les interfaces graphiques nous avons utilisé Apache NetBeans.
Uwamp qui permet de faire tourner un serveur web.
Les langages utiliser pour ce projet sont : JAVA et SQL.

Authentification

La page d’authentification comporte deux champs :
○ Nom d’utilisateur.
○ Mot de passe en *.
Seule l’admin peut crée un compte.


Menu principal

Vous avez 3 choix possible:
Liste , Ajout , Dashboard.
Il y’a un bouton pour se déconnecter également un bouton home pour revenir au menu principal (cette page).


Liste

○ soit on liste les medicament.
○ soit le type de personne.
il y'a une bar de recherche pour trouver un medicament, on insere le nom dans le champs “entrer le nom d’un medicament” et on clique sur ‘rechercher’ ellle se trouve egalement pour le type-personne.
pour lister les pertubateurs on double clique sur le medicament et s’affiche sur le label.


pour la modification il faut cliquer sur le crayon


Ajout

il y'a plusieurs choix:
